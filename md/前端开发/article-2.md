Vue 3 带来的最大变化莫过于组合式 API（Composition API）的引入。它不仅仅是 Options API 的替代品，更是一种全新的代码组织方式。

## 为什么需要组合式 API？

在 Options API 中，同一个逻辑关注点的代码分散在 data、methods、computed 等不同选项中。当组件变得复杂时，理解和维护成本急剧上升。

## 核心优势

- 更好的代码组织
- 更灵活的逻辑复用
- 更完善的类型推导
- 更小的打包体积

组合式 API 让 Vue 开发者能够以更符合直觉的方式组织代码，将相关的逻辑放在一起，而不是按照选项类型分散。
